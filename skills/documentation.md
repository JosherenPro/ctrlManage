# Skill: Documentation

## Objective

Documenter la plateforme pour faciliter le développement et l'exploitation.

## Inputs

- Architecture
- Modules applicatifs
- Processus de déploiement

## Outputs

- Documentation technique
- Guides d'usage
- Références d'exploitation

## Tasks

1. Maintenir le `README`.
2. Documenter les décisions structurantes.
3. Décrire les flux critiques.
4. Synchroniser la doc avec le code.

## Deliverables

- `README.md`
- `docs/`
