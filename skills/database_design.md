# Skill: Database Design

## Objective

Modéliser un schéma de données robuste pour les cours et présences.

## Inputs

- Règles métier
- Rôles
- Flux de présence

## Outputs

- Modèle relationnel
- Contraintes d'intégrité
- Schéma Prisma

## Tasks

1. Identifier les entités.
2. Définir les relations.
3. Ajouter contraintes et index.
4. Formaliser le schéma Prisma.

## Deliverables

- `docs/database_schema.md`
- `prisma/schema.prisma`
