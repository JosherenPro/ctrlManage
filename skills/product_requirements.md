# Skill: Product Requirements Engineering

## Objective

Transformer les besoins métier en backlog clair et testable.

## Inputs

- Vision produit
- Parties prenantes
- Contraintes métier

## Outputs

- User stories
- Critères d'acceptation
- Priorisation MVP

## Tasks

1. Identifier les rôles et parcours.
2. Rédiger les user stories.
3. Définir les critères d'acceptation.
4. Prioriser le backlog.

## Deliverables

- `docs/requirements.md`
- `docs/user_stories.md`
