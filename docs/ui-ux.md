# UI / UX

## Principes

- Mobile first pour le parcours étudiant.
- Interfaces simples, peu de friction, forte lisibilité.
- Accessibilité minimale : contraste suffisant, navigation clavier, libellés explicites.

## Parcours critiques

### Étudiant

- Connexion
- Accès au tableau de bord
- Scan du QR code
- Consultation de l'historique

### Professeur

- Connexion
- Liste des cours
- Ouverture d'une session
- Affichage du QR code
- Validation des présences

### Admin

- Connexion
- Gestion des utilisateurs
- Supervision et rapports

## Écrans du MVP

- Page de connexion
- Dashboard par rôle
- Liste des cours
- Détail d'un cours
- Écran de scan QR
- Vue session et présences
- Rapports

## Recommandations design

- Utiliser un design system Tailwind avec tokens de couleur et composants communs.
- Prévoir des états vides, de chargement et d'erreur dès le départ.
- Limiter les actions principales à une ou deux par écran.
